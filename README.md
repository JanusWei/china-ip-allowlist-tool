# 中国 IP 白名单工具

[![Python 3.8+](https://img.shields.io/badge/Python-3.8%2B-3776AB?logo=python&logoColor=white)](https://www.python.org/)
[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)
[![Data: APNIC](https://img.shields.io/badge/Data-APNIC-0052CC)](https://ftp.apnic.net/stats/apnic/)

从 APNIC 官方最新数据生成中国大陆、中国香港、中国澳门、中国台湾的 IPv4/IPv6 CIDR 清单，用于边界防火墙地址对象、访问白名单和地址变化审计。

无需安装第三方 Python 包，不依赖固定盘符，也不保存防火墙账号密码。把项目复制到任意 Windows、Linux 或 macOS 电脑，只要安装了 Python 3.8 或更高版本即可运行。

- [下载最新便携版](https://github.com/JanusWei/china-ip-allowlist-tool/releases/latest)
- [查看当前地址结果](结果/当前版本)
- [查看更新记录](更新日志.md)

## 它能解决什么问题

如果互联网业务只希望允许中国大陆或指定中国地区访问，防火墙通常需要一份可靠、可持续更新的 CIDR 白名单。

这个工具会自动完成：

1. 从 APNIC 官网下载最新 delegated 数据和官方 MD5。
2. 校验下载文件、报告日期、记录数量和数据结构。
3. 提取 CN、HK、MO、TW 的 IPv4 和 IPv6 地址段。
4. 聚合重复或相邻网段，生成适合导入的纯文本文件。
5. 与上一版本比较，输出真正新增和删除的 CIDR。
6. 发现数据异常时停止替换，继续保留上一版本。

## 防火墙应该导入哪个文件

| 使用场景 | 推荐文件 |
|---|---|
| 只允许中国大陆 IPv4 | `CN_中国大陆_IPv4.txt` |
| 只允许中国大陆 IPv6 | `CN_中国大陆_IPv6.txt` |
| 允许中国大陆、香港、澳门、台湾 IPv4 | `CN_HK_MO_TW_全部_IPv4.txt` |
| 允许中国大陆、香港、澳门、台湾 IPv6 | `CN_HK_MO_TW_全部_IPv6.txt` |
| 设备明确支持 IPv4/IPv6 混合导入 | `CN_HK_MO_TW_全部_IPv4和IPv6.txt` |

多数防火墙需要分别创建 IPv4 和 IPv6 地址对象，因此优先使用两个分开的总文件。所有导入文件均为“每行一个 CIDR”，没有标题和注释。

当前版本还提供每个地区的独立文件：

```text

CN_中国大陆_IPv6.txt
HK_中国香港_IPv4.txt
HK_中国香港_IPv6.txt
MO_中国澳门_IPv4.txt
MO_中国澳门_IPv6.txt
TW_中国台湾_IPv4.txt
TW_中国台湾_IPv6.txt
```

## 数据为什么可信

工具默认使用以下 APNIC 官方来源：

- 数据文件：<https://ftp.apnic.net/stats/apnic/delegated-apnic-latest>
- 官方 MD5：<https://ftp.apnic.net/stats/apnic/delegated-apnic-latest.md5>
- 格式规范：<https://www.apnic.net/about-apnic/corporate-documents/documents/resource-guidelines/rir-statistics-exchange-format/>

每次运行会执行以下校验：

- APNIC 官方 MD5 与本地文件一致。
- 文件头总记录数与实际记录数一致。
- ASN、IPv4、IPv6 分类数量与汇总记录一致。
- 报告日期不超过配置的新鲜度门限。
- 所有输出均为规范公网 CIDR。
- 各地区数量不低于安全下限。
- 地区文件、IPv4 总文件、IPv6 总文件和双栈总文件相互一致。
- 所有结果文件记录 SHA-256。
- 删除地址比例超过 5% 时停止提升新版本。

完整校验信息保存在：

```text
结果/当前版本/校验清单.json
```

## 如何查看新增和删除地址

每次运行都会将新数据与上一版本按“实际地址空间”比较，而不是简单比较 CIDR 字符串。

差异文件位于：

```text
结果/当前版本/差异/
```

其中包含：

- 每个地区 IPv4/IPv6 的新增和删除文件。
- 四地区 IPv4/IPv6 总合集的新增和删除文件。
- `变更摘要.json`，方便其他程序读取。

第一次运行没有上一版本，因此所有地址会显示为新增。从第二次运行开始，才是相对上一版本的真实变化。

## 演练、回滚和安全门限

只下载、校验并生成候选版本，不替换当前版本：

```bash
python 一键运行.py --演练
```

目录用途：

```text
结果/
├─ 当前版本/    当前可使用的地址文件
├─ 历史版本/    最近 3 个旧版本，供回滚
└─ 候选版本/    演练或未通过安全门限的数据
```

正常更新成功后会自动清理临时候选文件。校验失败、数据数量异常或删除变化过大时，当前版本不会被覆盖。

## 重要限制

APNIC 的国家代码表示地址资源首次分配或登记的国家/地区，并不等同于实时、精确的终端地理位置。

生产上线前应当：

- 先观察防火墙命中日志，再启用正式拒绝。
- 为合法境外合作方、监控节点、支付回调、CDN/WAF 回源地址建立例外。
- 同时处理 IPv4 和 IPv6，避免只封禁 IPv4 后被 IPv6 绕过。
- 仅将地域限制应用到指定业务地址和端口，不要误用于防火墙管理面或 VPN。

如果设备自带持续更新的“中国大陆”地域地址库，并且精度满足业务要求，应优先评估设备内置能力，本工具可作为校验、审计和应急来源。

## 常见问题

### 是否需要安装 `requests`、`pandas` 等第三方库？

不需要。项目只使用 Python 标准库。

### 可以直接双击运行吗？

如果操作系统已经把 `.py` 文件关联到 Python，可以双击；为了看到完整执行结果，更推荐在终端执行 `python 一键运行.py`。

### 为什么第一次运行显示全部地址都是新增？

因为第一次运行没有上一版本作为比较基准，这是正常现象。

### 为什么不直接生成“境外黑名单”？

业务目标是只允许指定中国地区访问时，“允许清单 + 其他来源默认拒绝”更直接，也更不容易遗漏新出现的境外网段。

### 运行失败会不会把当前文件清空？

不会。下载、MD5、结构、日期、数量或差异门限任意一项失败时，当前版本保持不变。

## 退出码

| 退出码 | 含义 |
|---:|---|
| `0` | 执行成功 |
| `1` | 下载、MD5、数据结构、日期或文件一致性校验失败 |
| `2` | 数量或大比例删除安全门限未通过，当前版本保持不变 |
