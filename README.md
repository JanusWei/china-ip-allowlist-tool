# 中国 IP 白名单工具

这是一个无需安装第三方依赖的便携工具。把整个文件夹复制到任意电脑，只要安装了 Python 3.8 或更高版本并且能够访问 APNIC 官网，就可以运行。

## 数据来源和适用边界

- 官方数据：https://ftp.apnic.net/stats/apnic/delegated-apnic-latest
- 官方 MD5：https://ftp.apnic.net/stats/apnic/delegated-apnic-latest.md5
- 格式说明：https://www.apnic.net/about-apnic/corporate-documents/documents/resource-guidelines/rir-statistics-exchange-format/

APNIC 国家代码表示地址资源首次分配或登记的国家/地区，不等同于实时、精确的终端地理位置。上线防火墙白名单前，应先观察命中日志，并为合法的境外合作方、监控节点、CDN/WAF 回源地址保留例外。

## 一键运行

在本项目最外层执行：

```text
python 一键运行.py
```

Windows、Linux、macOS 使用相同命令。如果系统中的 Python 命令叫 `python3`，则执行：

```text
python3 一键运行.py
```

只测试、不替换当前版本：

```text
python 一键运行.py --演练
```

防火墙一般需要分别导入 IPv4 和 IPv6，请优先使用两个分开的总文件。只有设备明确支持在同一个地址对象中混合 IPv4 和 IPv6 时，才使用双栈总文件。

## 目录说明

```text
中国IP白名单工具/
├─ 一键运行.py           只需运行这个文件
├─ README.md             使用说明
├─ 程序/                 内部执行程序，不需要手工修改
├─ 配置/                 APNIC 地址、地区和安全门限
├─ 结果/
│  ├─ 当前版本/          当前可以使用的文件
│  ├─ 历史版本/          上一个版本，供回滚
│  └─ 候选版本/          演练或未通过安全门限的版本
└─ 日志/                 每次运行的中文日志
```

`结果` 和 `日志` 会在第一次运行时自动创建。

工具最多保留 3 个历史版本和 10 份运行日志，避免长期运行后目录无限增长。正常更新使用的候选文件会在成功后自动清理；演练或校验失败的候选文件会保留，方便检查。

## 数据校验

每次运行都会自动完成：

1. 通过 HTTPS 下载 APNIC 官方最新数据和官方 MD5。
2. 比较官方 MD5 与本地下载文件。
3. 检查 APNIC 文件头、报告日期、总记录数和分类记录数。
4. 检查 CN、HK、MO、TW 的 IPv4/IPv6 地址是否为规范公网 CIDR。
5. 检查输出数量是否低于安全下限。
6. 计算所有输出文件的 SHA-256。
7. 反向检查地区文件、IPv4 总文件、IPv6 总文件和双栈总文件是否一致。
8. 与上一版本比较真正新增和删除的地址空间。
9. 删除地址超过 5% 时停止替换当前版本。

任何关键校验失败，工具都会保留上一版本，不会用错误或空文件覆盖当前结果。

## 新增和删除文件

`结果/当前版本/差异` 中包含各地区、各 IP 版本和总合集的新增、删除 CIDR。

第一次运行没有上一版本，因此所有地址都会显示为新增。以后再次运行时，才会得到相对上一版本的真实变化。

## 修改配置

普通使用不需要修改配置。需要调整数据新鲜度、删除比例或最低数量时，编辑 `配置/配置.json`。

配置文件使用中文字段，并且所有路径都相对于本项目自身目录，不包含任何固定盘符、用户名或开发电脑地址。

## 退出码

- `0`：执行成功。
- `1`：下载、MD5、数据结构、日期或文件一致性校验失败。
- `2`：数量或大比例删除安全门限未通过，当前版本保持不变。
